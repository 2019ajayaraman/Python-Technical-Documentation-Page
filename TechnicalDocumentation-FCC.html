    Boolean

    To understand basic python code, it is important to first understand boolean operators. Fortunately, rather than using &&, !, and || like in Java, python is meant to be a readable language so it uses and, not, and or respectively. Additionally, our boolean values are capitalized. Let's take a look at some examples:

    b1 = True
    b2 = False

    We see that b1 is set to the value True and b2 is set to the value False. Now let's take a look at how why can apply operations on these values to create different boolean values.

    b1 and b2               #True and False --> False
    b1 or b2                #True or False --> True
    not b1                  #not True --> False
    not (b2 or (b1 and b2)) #False

    Study the rules above and you will be able to master boolean logic in no time. It just takes a bit of practice. To recap, the boolean operators that will be most important for python beginners are:

  • and
  • or
  • not

    • Understanding boolean logic will be very important for the next section so make sure you understand it before continuing.

    Loops

    As stated above, loops NEED boolean logic to work. This logic allows the loops to understand how many times and when to repeat. We will be dealing with while and for loops in this section.

    While-Loops

    While-loops are the easier of the two. With simple syntax, all you have to write is 'while' and the boolean condition that you would like it to satisfy. When iterating with a while loop, be sure to increment the value within the body of the loop so you do not end up with an infinite loop. Let's take a look

    increm = 0 while increm < 5:
      print increm
      increm += 1

    This code prints the values of 0 through 5 because it iterates through the loop until iter is no loner less than 5. In other words, the code exits the loop once the value of iter becomes 5. Let's take a look at another example.

    while True:
      print "Hello World!"

    This code prints "Hellow World" an infinte number of times because the condition is always True. If we wanted to exit the loop, we would have to put a break inside the while-loop.


    For-Loops

    It is no suprise that for-loops are harder to understand in comparison to while-loops. The discrepancy in difficulties is due to how we must setup the boolean condition within the for-loop. First, we must define a for-loop. A for-loop is a loop that increments a fixed number of times. Rather than satisfying any condition, the for-loop is used when you know how many times you want to repeat a process. With for-loops, we will be using the range() and the in functions.

    The range() function has three parameters: starting value, ending value (exclusive), and step, in that order. Let's take a look at how this works.

    for i in range(0, 7, 2):
      print i

    Did you notice how we added the in function? Essentially, on each turn of the loop, the varaible i is set to the sequential number in the range, which in this case is 1-6 since the ending number is exclusive. However, since we set the step = 2, we added an additional qualification: every other number is printed. This means that i equals 0, 2, 4, and 6 depending on which iteration of the loop the code is on.

    To recap, we discussed the following concepts about loops:

  • While-loops
    • break
  • For-loops
    • range (start, end, step)
    • in

    • Loops are the meat of coding so now that we have gotten that out of the way, we can talk about a bit more advanced principles.

    Files

    When talking about files, we are talking about text files that we can read data from, that we can create, or that we can edit using python.

    To start off, we must open up the textfile we want. Please note, this textfile need not exist prior to executing the code. If it doesn't exist, your computer will create a blank textfile under the name you enter. However, this really only works when you want to create a textfile. Otherwise, your computer will throw an error if you attempt to read date AND do something with it as the data is nonexistent. To open a file, use the following code:

    file = open("filename.txt", setting)

    In the code above, we are opening a file with the name "filename" and the settings setting. It is important to include '.txt' with your filename to ensure that the code opens the correct document and does not throw an error. We will discuss settings more below.

    Reading

    With reading, the setting is 'r'. This gives the file object the functionality of being read using different methods. These methods include read(), readline()), and readlines().

  • .read() will return a large string of the entire document. To deal with smaller parts of this string, you would need to be able to parse the string into lines and then even further into words or even characters.
  • .readline()returns one line at a time. Every time you apply this method to the file object, a single line is returned. This object is best used with a while-loop since we don't know how many lines there will be in the document.
  • .readlines() returns a list of strings for each of the lines in the textfile. At each index, there is a differnt line. To iterate through all the lines using this method, a for-loop is best used because we know the length of the list already.

    • Let's take a look at all of the examples below:

    file = open("filename.txt", 'r')

    #printing file after reading with .read()
    readFile = file.read()
    print (readFile)

    #printing file after reading with .readline()
    readLine = file.readline()
    while readLine:                      #This test to see if the current line has any content so now error is thrown
      print (readLine)
      readLine = file.readLine()         #This iterates to the next line in the file

    #printing file after reading with .readlines()
    readLines = file.readlines()
    for line in readlines:   print (line)

    All of the code above accomplished the same task: printing out the entire textfile after reading, or iterating, through it. Let's move on to writing text files.

    Writing

    Let's say that you have to save the output your code submits on a textfile. To do this, we must first set the stting to 'w' for write. Unlike with reading, there is really only once simple method for writing. Also note that if you use a preexisting textfile, this function will write over any data already stored. This means you will lose any past data you might need. Check before attempting to use this function.

    The only function that will be used is the .write() function, which overwrites the entire document. Let's take a look:

    file = open("filename.txt", 'w')

    file.write("I am writing this to filename.txt")

    If you were to run the code above and then open filename.txt, you would notice that the entire document would contain the line "I am writing this to filename.txt".

    Now let's take a look at our final subsection for files: editing.

    Editing

    In the previous subsection, we talked about writing to files and how they overwrite text. But what if we wanted to only add text? We can do this by using the append function, with which the setting must be set to 'a'. This will allow us to keep prior data while also recording new data.

    When we open the file in this manner (with 'a'), we can just use the same .write() function as above.


    Closing document are equally as imporant as opening them. If you do not close a document, you migth get a later on, generally due to the overload of data in the server. After you have processed all the data needed, add the line file.close() to close the textfile.

  • Reading files
    • .read()

  • Writing files
  • Editing (or appending) files

    Running

    We have reached our final section! Running code is actually quite simple. Unlike programming languages like Java, there is no need to create any methods, although you could. You simply must just write anything you want to run in the file, click run, and watch your code work its magic.

    However, this method is only best when you are using simple code. With complex code and large projects, it is best to keep all of your code in methods. In this case, use the following code:

    if __name__ == "__main__":
      main()

    This code above tests to see if the program is running and if so, allows the program to call main(), another method that must be initialized before running the code. To so, simply enter main(): and start to enter code in an indented fashion underneath it. Be sure, however, that the entire main() method is above the call to main(). It helps with running and debugging if this is the case.
